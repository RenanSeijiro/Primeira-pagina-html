Comunicação Client x Server

Primeira coisa que precisamos entender quando falamos de uma pagina web, é que basicamente todo site fica dentro de um servidor local e nos acessamos ele atravez da internet. Ou seja, o Client conectado a internet, solicita acesso ao site que está hospedado em um Server, atravez de um link.

Mas afinal o que significa esses termos?

  • Client

    • Esse é o lado do usuario final. são os dispositivos que possuem softwares como navegadores, capazes de se conectar a internet para acessar um site. esse lado é necessario ter uma conexão boa com a internet. Caso contrario a possa ser que demore para carregar uma pagina web ou até mesmo sofrendo algumas engasgadas durante o acesso.

    Exitem duas principais formas de acessarmos um site, sendo elas:

    1. Aplicações Web
      2.

      Acesso atraves de navegadores. Sem a necessidade de intalações de programas para conexão.

    2. Aplicações não Web
      3.

      Que são basicamente o acesso atravez de software/aplicatios como facebook, instagram. Nesse casos os aplicativos já vem configurado para acessarem o servidor.

  • Server

    • Traduzido para o português "servidor", faz exatamente o que o nome sugere. Ele é uma maquina responsavel por servir dados a outras maquinas. Ou seja, superficialmente falando, é o local onde você guardar todos os arquivos, paginas, imagens, videos etc. Todo tipo de "dados" que você gostaria de deixar acessivel a outra maquina.

    Diferentemente do lado Client, no servidor utilizamos maquinas mais "parrudas". Maquinas que possuem componentes com capacidade de processamento alto. Uma conexão estavel é essencial tambem! Por conta disso muitas vezes utilizamos metodos de redundancia, com mais de uma provedora de internet. Garantindo a conexão caso a outra venha a cair.

    Eles podem ser acessados tanto atravez de uma conexão de rede local, como conexão via Web utilizando a internet. E essa conexão de outras maquinas com o servidor via internet, que chamamos de Comunicação client x Server!

HTML

HTML (Linguagem de Marcação de HiperTexto) é o bloco de construção mais básico da web. Define o significado e a estrutura do conteúdo da web.Outras tecnologias além do HTML geralmente são usadas para descrever a aparência/apresentação (CSS) ou a funcionalidade/comportamento (JavaScript) de uma página da web.

"Hipertexto" refere-se aos links que conectam páginas da Web entre si, seja dentro de um único site ou entre sites. Links são um aspecto fundamental da web. Ao carregar conteúdo na Internet e vinculá-lo a páginas criadas por outras pessoas, você se torna um participante ativo na world wide web.

O que são Tag's?

Tags são elementos responsaveis por informar o navegador que tipo de estrutura está sendo construida.

Por exemplo, a tag body informa ao nevegador que a partir da abertura até o fechamento dessa tag, será inserido dados no corpo de nossa pagina. E a tag p diz ao navegador que naquele momento se inicia um paragrafo de texto e passa as configurações de um paragrafo ao navegador.

Basicamente o arquivo HTML é composto por diversas Tag's reponsaveis por passar instruções qual o navegador consegue executar. E dentro dessa tags, somos capazes que colocarmos propiedades capazes de mudar essa instruções. chamamos essas propiedades de Atributos.
